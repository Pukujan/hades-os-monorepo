Extension Design Spec

Scope: browser extension only. No Socials install card, no main-app key generation screen, no main-app download card.

Layout + spacing

  • Popup: 360–390px wide, 600–640px tall, optimized for quick capture/chat/approval entry.
  • Side panel: 420–462px wide mobile-style command surface with persistent nav.
  • Cards: 22px radius, 14–16px padding, 10–12px vertical gaps.
  • Buttons: pill/command buttons, 38px minimum height, strong labels with small helper text.

Safety / approval UX

  • Always say what Hades wants to do before approve/reject.
  • Show target surface: form, URL, destination, or file context.
  • Show sensitive-field handling: values hidden unless the user explicitly reveals them.
  • Separate Fill fields from Submit form.
  • Default for external actions is draft/prep, not send/submit.
  • Approval cards must remain calm but visually distinct with warning color.
